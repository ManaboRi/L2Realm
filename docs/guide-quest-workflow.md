# Как добавлять квесты в базу гайдов

Квесты в L2Realm должны быть не заглушками, а короткими рабочими маршрутами: игрок открыл страницу и понимает, к кому идти, кого убить, что собрать и где забрать награду.

Этот файл — источник правды по записи квестов. Если меняешь схему/правила в коде, обнови и здесь.

## Поля квеста (что именно заполняем)

Одна запись = одна строка в таблице `Guide` (`category: "quests"`). Ограничения — как в `backend/src/guides/guides.service.ts`.

| Поле | Что это | Ограничение / формат | Пример |
|---|---|---|---|
| `title` | Название **на русском** | ≤160 симв. Без англ. в скобках | `Покаяние` |
| `titleEn` | Англ. название (рендерится подзаголовком под русским) | ≤160 симв., можно пусто | `Repent Your Sins` |
| `slug` | URL-хвост | англ., авто из title если пусто; уникальность сама (`-2`, `-3`) | `repent-your-sins` |
| `chronicle` | Хроника(и) | `interlude` / `high-five` / `interlude,high-five` / `all` | `interlude,high-five` |
| `category` | Раздел | всегда `quests` | `quests` |
| `levelMin` / `levelMax` | Диапазон уровней | целые или пусто | `18` / `40` |
| `npc` | Стартовый/ключевой NPC | ≤80 симв., имя как в базе NPC | `Чёрный Судья` |
| `location` | Базовый город/локация | ≤80 симв., сводим к городу | `Глудио` |
| `reward` | Награда строкой с иконками | **≤120 симв.** (длиннее — обрежется!) | `228,064 :exp: · 16,455 :sp: · Leaf of Oracle` |
| `types` | Теги-типы (фильтр + сортировка) | массив из точных меток (см. ниже) | `["Профессия","Разовые"]` |
| `repeatable` | Повторяемый ∞ | bool, **согласован с тегом** Повторяемые/Разовые | `false` |
| `description` | SEO-описание (meta) | ≤320 симв., целимся в 120–160 | `Квест «Покаяние»: снижаем PK через пета Sin Eater.` |
| `content` | Тело страницы (Markdown) | без лимита | см. «Формат контента» |
| `image` | Обложка/портрет (опц.) | URL или пусто | `/images/...` |
| `published` | Публиковать | bool (иначе черновик) | `true` |
| `sort` | Ручной порядок | целое, по умолчанию 0 | `0` |

## Источники и факты

Для каждого квеста сначала собираем факты: рус/англ название, уровень и диапазон, стартовый/ключевой NPC, локация, награды (EXP/SP/адена + предметы), тип, хроника появления и удаления, пошаговый маршрут.

Текст прохождения пишем **своими словами**. Берём факты, а не формулировки источника. Не оставляем заглушки вроде «проверьте журнал квестов» или «выполните поручение» без конкретики.

Если источник отдаёт длинную кашу из вкладок, расовых развилок, вопросов теста или технических стадий — **не пускаем её через общий шаблон**. Делаем ручной override: короткое описание, требования, аккуратный маршрут, награда и связанные сущности. Особенно важно для квестов с расовыми развилками, крафтом или Территориальными Войнами.

## Хроники

Для классической базы учитываем `Interlude` и `High Five`. Поле `chronicle` хранит одну метку или несколько через запятую (`interlude,high-five`); фильтр matchает по вхождению, `all` = показывать во всех.

Порядок патчей для проверки появления/удаления:

1. Prelude · 2. Chronicle 1 · 3. Chronicle 2 · 4. Chronicle 3 · 5. Chronicle 4 · 6. Chronicle 5 · 7. Interlude · 8. The Kamael · 9. Hellbound · 10. Gracia Part 1 · 11. Gracia Part 2 · 12. Gracia Final · 13. Gracia Epilogue · 14. Freya · 15. High Five · 16. Awakening · 17. Tauti · 18. Glory Days · 19. Lindvior · 20. Epeisodion · 21. Ertheia

Правила привязки:

- добавлен до/в `Interlude` и удалён после `High Five` → `interlude,high-five`;
- добавлен после `Interlude`, но до или в `High Five` → `high-five`;
- добавлен после `High Five` → оставляем черновиком до базы Main/Essence;
- удалён до нужной хроники → в этой хронике не показываем;
- строки «добавлен/удалён в обновлении» используем только для привязки; на публичной странице их не выводим.

## Типы (`types`)

Метки берём **дословно** — неточные молча отбрасываются (`normTypes`). Для квестов:

`Сюжетный` · `Профессия` · `Сабкласс` · `Дуал-Класс` · `Ноблес` · `Олимпиада` · `Клановые` · `Ежедневные` · `Повторяемые` · `Рейд-Босс` · `Питомцы` · `Разовые`

Правила:

- ставим осмысленно — по типам идёт фильтр и сортировка;
- ровно один из `Повторяемые` / `Разовые` и держим `repeatable` в соответствии (Повторяемые → `repeatable:true`, Разовые → `false`);
- можно несколько тегов (напр. `Клановые` + `Разовые`).

Полный список меток и цвета — `frontend/src/app/guides/questTypes.ts`.

## Награды (`reward`)

Формат — части через `·`, чтобы фронт показал несколько иконок:

```
228,064 :exp: · 16,455 :sp: · 163,800 :adena: · Leaf of Oracle
```

- `:exp:` `:sp:` `:adena:` → иконки. В списке показываются **только иконки** (без цифр), на детальной — «Опыт (228,064)» и т.д.;
- **предметы пишем точным англ. названием.** Если оно есть в карте иконок (`frontend/src/app/guides/reward.tsx` → `ITEM_REWARD_ICONS`, файлы в `frontend/public/images/WIKI`) — отрисуется иконкой; иначе текстом со значком-свитком;
- если иконки предмета ещё нет — либо добавить картинку в `WIKI` и метку в карту, либо оставить текст;
- **держим ≤120 символов** — берём EXP/SP/адену + 1–2 ключевых предмета, без длинных списков временного дропа;
- `reward` и блок `## Награда` в теле должны совпадать.

## Формат контента (`content`, Markdown)

Порядок блоков на странице:

1. `## Что это`
2. `## Требования`
3. `## Прохождение`
4. `## Награда`
5. `## Связанные NPC и предметы`

Шаги прохождения — списком, каждый отвечает хотя бы на один практический вопрос (куда идти / кого бить / что собрать / кому сдать):

```md
1. Поговорите с NPC в нужной локации.
2. Убейте нужных монстров и соберите предметы.
3. Вернитесь к NPC и получите следующий предмет или награду.
```

В тексте можно встраивать иконки наград инлайн: `получите 50,000 :adena:`.

**Разворачивающийся блок** (для расовых развилок, необязательных деталей, длинных списков дропа) — нативный `<details>`, без JS:

```md
::: details Варианты по расам
- **Человек:** ...
- **Эльф:** ...
:::
```

Внутри блока работает обычный Markdown (списки, шаги, иконки). Не прячем в него основной маршрут — только дополнительное.

## Картинка (`image`)

Необязательна. Если есть — рендерится справа от заголовка на детальной; если нет — заголовок на всю ширину (не «ломается»). План: подставлять портрет стартового NPC (у NPC картинки уже есть). Списки квестов сейчас без превью — это норма.

## Локации

В `location` ставим только основной город/базовую локацию для навигации. «Замок Гиран» / «Земли Орена» сводим к `Гиран` / `Орен`. Для Gludio-пакета — `Глудио`. Не плодим декоративные «замки» и «земли».

## Связи NPC, предметов и квестов

Фронт сам превращает совпадение имени в ссылку (`buildAutoLinks`). Поэтому **связанные NPC, монстры, предметы и локации пишем теми же именами**, что есть/будут в базе. На странице NPC упоминаем название квеста в блоке `## Связанные квесты` (пример: `Рабочий Склада Хапрок` ↔ `Странное родство`).

## Способы добавления

1. **Seed-пачка по городу** — основной путь для больших партий. Отдельный файл `seed-<город>-quests.js` (`node prisma/seed-gludio-quests.js`), парсит факты + генерит markdown + делает upsert. Удобно перезапускать только нужный город.
2. **Админка** — для одиночных правок: все поля, чипсы типов, загрузка картинки.
3. **Bulk API** — `POST /api/proxy/guides` с Bearer (key-login) для скриптовых партий без отдельного seed-файла. Тело слать **UTF-8 байтами** (иначе кириллица бьётся в `?`).

Перед добавлением **проверяем, нет ли квеста в базе** (по slug/title) — чтобы не плодить дубли. Seed обновляет существующий по `slug`, сохраняя его (не ломаем публичные URL и SEO).

## Пачки по городам

Крупные партии держим отдельными seed-файлами по городу/смыслу (`seed-gludio-quests.js`, `seed-gludin-quests.js`). Если квест уже есть — seed обновляет текст, награды, хроники и связи, но **сохраняет существующий `slug`**. Новый `slug` — только для новых страниц.

## URL

Английский `slug` по названию квеста, без города и хвостов:

- `a-clans-reputation`
- `enhance-your-weapon`
- `possessor-of-a-precious-soul-1`

Город в URL не пишем (он в `location`). Старые опубликованные URL массово не переименовываем — при необходимости сначала редиректы + проверка canonical/sitemap, иначе 404 и потеря SEO.

## Самопроверка перед публикацией

После добавления/обновления открываем итоговую страницу и список, проверяем не только факты, но и как читается:

- в начале нет случайного куска диалога/таблицы/вкладки источника;
- `## Прохождение` начинается с понятного первого действия (к кому, где, что взять);
- каждый шаг отвечает на практический вопрос; длинные цепочки разбиты на короткие шаги;
- нет заглушек («проверьте журнал квестов», «выполните поручение» без конкретики);
- мусорные переводы предметов из источника удалены;
- награда в `reward` и в блоке `## Награда` совпадает, EXP/SP/адена — отдельными частями, иконки наград и предметов отрисовались;
- теги `types` показались, фильтр по хронике и сортировка работают;
- связанные NPC/предметы/локации стали ссылками (имена совпали с базой);
- `title` без латиницы, англ. ушло в `titleEn`; `description` заполнено (120–160).
