# Как добавлять монстров в базу гайдов

Монстры в L2Realm нужны не ради количества, а ради связей: игрок читает квест, кликает по имени моба и сразу видит уровень, агрессию, локацию, дроп, спойл и связанные задания.

## Поля монстра

Одна запись = одна строка `Guide` с `category: "monsters"`.

| Поле | Правило |
|---|---|
| `title` | Русское название монстра. |
| `titleEn` | Английское название как в игре/источнике. |
| `slug` | Английский URL по `titleEn`, без города и лишних хвостов. |
| `chronicle` | Для классики используем `interlude`, `high-five` или `interlude,high-five`; Main/Essence отдельно. |
| `levelMin` / `levelMax` | Уровень монстра. Если уровень один, заполняем `levelMin`, `levelMax` оставляем пустым. |
| `location` | Основная зона охоты: `Море Спор`, `Лагерь Ящеров Мэль`, `Кладбище`, `Поле Брани`. Для монстров не сводим всё к городу, игроку нужна точная зона. |
| `reward` | Короткая строка ключевого дропа для списка: `:adena: · Charcoal · Steel`. Длинный дроп уходит в Markdown-таблицу. |
| `types` | Минимум `Монстр`. Добавляем `Агрессивный`, `Социальный`, `Квестовый монстр`, `Дроп`, `Спойл` только если это подтверждено фактами. |
| `image` | Скрин/портрет монстра. Если точной картинки нет, лучше оставить пусто, чем ставить случайную. |

## Формат страницы

Порядок блоков:

1. `## Что это`
2. `## Где находится`
3. `## Дроп`
4. `## Спойл`
5. `## Используется в квестах`
6. `## Связанные локации и предметы`

Дроп и спойл оформляем таблицами:

```md
| Предмет | Кол-во | Шанс |
|---|---:|---:|
| Adena | 512-1198 | 70% |
| Charcoal | 1 | 3.65% |
```

Для `reward` берём 3-4 самые полезные позиции, без длинного хвоста хербов. Внутри страницы можно показывать больше строк.

## Связи

Имена монстров в квестах пишем так же, как `title` или `titleEn` у страницы монстра. Тогда автоссылки сами превратят текст в кликабельную ссылку.

Если у монстра есть квестовое применение, в контенте обязательно добавляем блок:

```md
## Используется в квестах

- **Repent Your Sins**
- **In Search of Cloth**
```

## Самопроверка

Перед публикацией проверяем:

- русское и английское название не смешаны в одном поле;
- уровень, агрессия, HP/EXP/SP и локация читаются в первые 5 секунд;
- таблица дропа не раздута мусором: топовые строки сверху, хербы не забивают всё полезное;
- спойл отделён от дропа;
- связанные квесты и локации написаны теми же именами, что есть в базе;
- картинка не растягивает шапку страницы и не превращается в мутный баннер;
- если данных по спойлу нет, пишем честно: `Нет проверенных данных по спойлу.`
